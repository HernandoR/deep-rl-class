# Huggy 的工作原理 [[how-huggy-works]]

Huggy是由Hugging Face制作并基于 [Puppo the Corgi，Unity MLAgents 团队的项目](https://blog.unity.com/technology/puppo-the-corgi-cuteness-overload-with-the-unity-ml-agents-toolkit).
该环境是使用创建的 [Unity游戏引擎](https://unity.com/) 和 [MLAgents](https://github.com/Unity-Technologies/ml-agents)。 ML-Agents 是 Unity 游戏引擎的工具包，它使我们能够 **使用 Unity 创建环境或使用预制环境来培训我们的代理**.

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/notebooks/unit-bonus1/huggy.jpg" alt="Huggy" width="100%">

在这个环境中，我们的目标是训练 Huggy **拿起我们扔的棍子。这意味着他需要正确地朝棍子移动**.

## 状态空间，Huggy 感知的东西。 [[state-space]]
Huggy 没有“看到”他的环境。相反，我们向他提供有关环境的信息：

- 目标 (棍子) 位置
- 自己与目标的相对位置
- 他的腿的方向。

考虑到所有这些信息，Huggy 可以 **使用他的政策来确定下一步要采取哪些行动来实现他的目标**.

## 动作空间，Huggy可以进行什么动作 [[action-space]]
<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/notebooks/unit-bonus1/huggy-action.jpg" alt="Huggy action" width="100%">

**关节电机驱动Huggy的腿**。这意味着要获得目标，Huggy 需要**学习正确旋转每条腿的关节电机，以便他可以移动**.

## 奖励函数 [[reward-function]]

奖励函数的设计使得 **Huggy将实现他的目标**: 拿棍子来。

请记住，强化学习的基础之一是 *奖励假设*：目标可以描述为 **期望累积奖励最大化**.

在这里，我们的目标是 Huggy **朝棍子方向移动，但不会旋转太多**。因此，我们的奖励函数必须转化这个目标。

我们的奖励函数：

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/notebooks/unit-bonus1/reward.jpg" alt="Huggy reward function" width="100%">

- *迎新奖金*：我们 **奖励他接近目标**.
- *时间处罚*：每次行动都会受到固定时间的惩罚 **迫使他尽快到达棍子处**.
- *轮换惩罚*：我们惩罚 Huggy 如果 **他旋转太多并且转身太快**.
- *达到目标奖励*：我们奖励 Huggy **达到目标**.

如果你想看看这个奖励函数在数学上是什么样子，请检查 [柯基犬 Puppo 介绍](https://blog.unity.com/technology/puppo-the-corgi-cuteness-overload-with-the-unity-ml-agents-toolkit).

## 火车 Huggy

Huggy 目标 **学会正确地、尽可能快地跑向目标**。为此，在每一步并根据环境观察，他需要决定如何旋转腿部的每个关节电机以正确移动 (不要旋转太多) 并朝着目标前进。

这是一个地方 **棍子是随机产生的**。当 Huggy 到达它时，棍子会在其他地方生成。
