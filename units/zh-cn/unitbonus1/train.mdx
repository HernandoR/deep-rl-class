## 让我们回顾一下这个环境是如何运作的

### 状态空间：Huggy 感知的内容。

Huggy 没有“看到”他的环境。相反，我们向他提供有关环境的信息：

- 目标 (棍子) 位置
- 自己与目标的相对位置
- 他的腿的方向。

鉴于所有这些信息，Huggy **可以决定下一步采取什么行动来实现他的目标**.

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/notebooks/unit-bonus1/huggy.jpg" alt="Huggy" width="100%">


### 行动空间：Huggy 可以做什么
<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/notebooks/unit-bonus1/huggy-action.jpg" alt="Huggy action" width="100%">

**关节电机驱动拥抱腿**。这意味着要获得目标，Huggy 需要 **学习正确旋转每条腿的关节电机，以便他可以移动**.

### 奖励函数

奖励函数的设计使得**Huggy将实现他的目标** : 拿棍子来。

请记住，强化学习的基础之一是 *奖励假设*：目标可以描述为 **期望累积奖励最大化**.

在这里，我们的目标是 Huggy **朝棍子方向移动，但不会旋转太多**。因此，我们的奖励函数必须转化这个目标。

我们的奖励函数：

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/notebooks/unit-bonus1/reward.jpg" alt="Huggy reward function" width="100%">

- *迎新奖金*：我们 **奖励他接近目标**.
- *时间处罚*：每次行动都会受到固定时间的惩罚 **迫使他尽快到达棍子处**.
- *轮换惩罚*：我们惩罚 Huggy 如果 **他旋转太多并且转身太快**.
- *达到目标奖励*：我们奖励 Huggy **达到目标**.

## 检查 Huggy 配置文件

- 在 ML-Agents 中，您定义 **在 config.yaml 文件中训练超参数。**

- 对于本笔记本的范围，我们不会修改超参数，但如果您想尝试作为实验，Unity 提供了非常 [很好的文档在这里解释了它们中的每一个](https://github.com/Unity-Technologies/ml-agents/blob/main/docs/Training-Configuration-File.md).

- 我们需要为 Huggy 创建一个配置文件。 

- 前往 `/content/ml-agents/config/ppo`

- 创建一个名为的新文件 `Huggy.yaml`

- 复制并粘贴以下内容🔽

```
behaviors:
  Huggy:
    trainer_type: ppo
    hyperparameters:
      batch_size: 2048
      buffer_size: 20480
      learning_rate: 0.0003
      beta: 0.005
      epsilon: 0.2
      lambd: 0.95
      num_epoch: 3
      learning_rate_schedule: linear
    network_settings:
      normalize: true
      hidden_units: 512
      num_layers: 3
      vis_encode_type: simple
    reward_signals:
      extrinsic:
        gamma: 0.995
        strength: 1.0
    checkpoint_interval: 200000
    keep_checkpoints: 15
    max_steps: 2e6
    time_horizon: 1000
    summary_freq: 50000
```

- 不要忘记保存文件！

- **如果您想修改超参数**，在Google Colab笔记本中，您可以点击此处打开config.yaml： `/content/ml-agents/config/ppo/Huggy.yaml`

我们现在准备好训练我们的代理了🔥。

## 培训我们的代理商

为了训练我们的代理，我们只需要 **启动 mlagents-learn 并选择包含环境的可执行文件。**

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/notebooks/unit-bonus1/mllearn.png" alt="ml learn function" width="100%">

通过 ML Agent，我们运行训练脚本。我们定义四个参数：

1. `mlagents-learn <config>`：超参数配置文件所在路径。
2. `--env`：环境可执行文件所在的位置。
3. `--run-id`：您要为训练运行 ID 指定的名称。
4. `--no-graphics`：在训练期间不启动可视化。

训练模型并使用 `--resume` 标记在中断时继续训练。

> 第一次使用时会失败 `--resume`，尝试再次运行该块以绕过错误。



培训将需要 30 到 45 分钟，具体取决于您的机器 (别忘了 **设置 GPU**)，去拿一个☕️你值得拥有🤗。

```bash
mlagents-learn ./config/ppo/Huggy.yaml --env=./trained-envs-executables/linux/Huggy/Huggy --run-id="Huggy" --no-graphics
```
