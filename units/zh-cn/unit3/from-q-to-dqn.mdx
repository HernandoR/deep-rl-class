# 从 Q-Learning 到 Deep Q-Learning [[from-q-to-dqn]]

我们了解到 **Q-Learning 是我们用来训练 Q 函数的算法**, 一个 **行动价值函数** 它决定了处于特定状态并在该状态下采取特定行动的价值。

<figure>
  <img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/unit3/Q-function.jpg" alt="Q-function"/>
</figure>

的 **Q 来自该状态下该动作的“质量”。**

在内部，我们的 Q 函数编码为 **Q-table，每个单元格对应一个状态-动作对值的表。** 将此 Q 表视为 **我们的 Q 函数的内存或备忘单。**

问题是 Q-Learning 是 *表格法*。如果状态和动作空间 **不够小，无法用数组和表有效表示**。换句话说：它是 **不可扩展**.
Q-Learning 适用于小型状态空间环境，例如：

- FrozenLake，我们有 16 个状态。
- Taxi-v3，我们有 500 个状态。

但想想我们今天要做的事情：我们将训练一个代理来学习玩 Space Invaders，这是一个更复杂的游戏，使用帧作为输入。

作为 **[尼基塔·梅尔科泽洛夫提到](https://twitter.com/meln1k), Atari 环境** 有一个形状为的观察空间 (210, 160, 3)*，包含范围从 0 到 255 的值，这样我们就可以得到\\(256^{210 \times 160 \times 3} = 256^{100800}\\)可能的观察结果 (为了进行比较，我们大约有 \\(10^{80}\\) 可观测宇宙中的原子).

* Atari 中的单个帧由 210x160 像素的图像组成。鉴于图像是彩色的 (RGB)，有3个通道。这就是为什么形状是 (210, 160, 3)。对于每个像素，该值可以从 0 到 255。

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/unit4/atari.jpg" alt="Atari State Space"/>

因此，状态空间是巨大的；因此，为该环境创建和更新 Q 表效率不高。在这种情况下，最好的想法是使用参数化 Q 函数来近似 Q 值  \\(Q_{\theta}(s,a)\\)  .

给定一个状态，该神经网络将近似该状态下每个可能动作的不同 Q 值。这正是 Deep Q-Learning 所做的。

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/unit1/deep.jpg" alt="Deep Q Learning"/>


现在我们了解了 Deep Q-Learning，让我们更深入地了解 Deep Q-Network。
