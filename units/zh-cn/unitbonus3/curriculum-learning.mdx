# (自动) RL 课程学习

虽然本课程中看到的大多数 RL 方法在实践中效果良好，但在某些情况下单独使用它们会失败。例如，在以下情况下可能会发生这种情况：

- 学习任务是艰巨的，需要 **逐步获得技能** (例如，当一个人想让一个双足智能体学会穿过困难的障碍时，它必须首先学会站立，然后行走，然后也许会跳跃……)
- 环境有变化 (影响难度的) 并且希望其代理人是 **稳健的** 对他们

在这种情况下，似乎需要向我们的 RL 代理提出不同的任务并组织它们，以便代理逐步获得技能。这种方法称为 **课程学习** 通常意味着手工设计的课程 (或按特定顺序组织的一组任务)。在实践中，例如，可以控制环境的生成、初始状态，或者使用自我对战并控制向 RL 代理提出的对手的级别。

由于设计这样的课程并不总是微不足道的，因此 **自动课程学习 (前交叉韧带) 提出设计学习创建此类任务组织的方法，以最大限度地提高 RL 代理的性能**。波特拉斯等人。建议将 ACL 定义为：

> …一系列机制，通过学习调整学习情境的选择以适应 RL 代理的能力，从而自动适应训练数据的分布。
>

例如，使用 OpenAI **域随机化** (他们对环境施加了随机变化) 让机械手解决魔方问题。
