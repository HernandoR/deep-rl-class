### sample-factory如何运作

sample-factory是其中之一 **社区可用的最优化的 RL 实现**.

它的工作原理是 **生成多个运行 rollout 工作线程、推理工作线程和学习工作线程的进程**.

的 *工人* **通过共享内存进行通信，降低了进程间的通信成本**.

的 *推出工人* 与环境互动并将观察结果发送到 *推理工作者*.

的 *推理工人* 查询策略的固定版本并 **将操作发送回 Rollout Worker**.

之后 *k* 推出工作的步骤向学习者发送经验轨迹， **它用来更新代理的策略网络**.

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/unit9/samplefactory.png" alt="Sample factory"/>

### Sample-factory 中的 Actor Critic 模型

Sample Factory 中的 Actor Critic 模型由三个组件组成：

- **编码器** - 处理输入观察结果 (图像、矢量) 并将它们映射到向量。这是您最有可能想要自定义的模型部分。
- **核心** - 来自一个或多个编码器的集成向量，可以选择在基于内存的代理中包含单层或多层 LSTM/GRU。
- **解码器** - 在计算策略和值输出之前，将附加层应用于模型核心的输出。

该图书馆被设计为自动支持任何观察和行动空间。用户可以轻松添加自己的自定义模型。您可以在以下位置了解更多信息 [文档](https://www.samplefactory.dev/03-customization/custom-models/#actor-critic-models-in-sample-factory).

## VizDoom

[VizDoom](https://vizdoom.cs.put.edu.pl/) 是一个 **Doom 引擎的开源 Python 接口**.

该库由波兰波兹南理工大学计算科学研究所的 Marek Wydmuch 和 Michal Kempka 于 2016 年创建。

该库使 **在许多场景中直接从屏幕像素训练代理**，包括团队死亡竞赛，如下面的视频所示。由于 ViZDoom 环境基于 90 年代创建的游戏，因此它可以在现代硬件上以加速速度运行， **让我们能够相当快地学习复杂的人工智能行为**.

该库包括以下功能：

- 多平台 (Linux、macOS、Windows),
- 适用于 Python 和 C++ 的 API，
- [OpenAI Gym](https://www.gymlibrary.dev/) 环境包装器
- 轻松创建自定义场景 (可视化编辑器、脚本语言和可用示例),
- 异步和同步单人游戏和多人游戏模式，
- 轻量级 (几 MB) 而且快 (同步模式下高达 7000 fps，单线程),
- 可定制的分辨率和渲染参数，
- 访问深度缓冲区 (3D视觉),
- 自动标记帧中可见的游戏对象，
- 访问音频缓冲区
- 访问角色/对象列表和地图几何图形，
- 离屏渲染和剧集录制，
- 异步模式下的时间缩放。

## 在sample-factory中设置Doom环境

```python
import functools

from sample_factory.algo.utils.context import global_model_factory
from sample_factory.cfg.arguments import parse_full_cfg, parse_sf_args
from sample_factory.envs.env_utils import register_env
from sample_factory.train import run_rl

from sf_examples.vizdoom.doom.doom_model import make_vizdoom_encoder
from sf_examples.vizdoom.doom.doom_params import add_doom_env_args, doom_override_defaults
from sf_examples.vizdoom.doom.doom_utils import DOOM_ENVS, make_doom_env_from_spec


# Registers all the ViZDoom environments
def register_vizdoom_envs():
    for env_spec in DOOM_ENVS:
        make_env_func = functools.partial(make_doom_env_from_spec, env_spec)
        register_env(env_spec.name, make_env_func)


# Sample Factory allows the registration of a custom Neural Network architecture
# See https://github.com/alex-petrenko/sample-factory/blob/master/sf_examples/vizdoom/doom/doom_model.py for more details
def register_vizdoom_models():
    global_model_factory().register_encoder_factory(make_vizdoom_encoder)


def register_vizdoom_components():
    register_vizdoom_envs()
    register_vizdoom_models()


# parse the command line args and create a config
def parse_vizdoom_cfg(argv=None, evaluation=False):
    parser, _ = parse_sf_args(argv=argv, evaluation=evaluation)
    # parameters specific to Doom envs
    add_doom_env_args(parser)
    # override Doom default values for algo parameters
    doom_override_defaults(parser)
    # second parsing pass yields the final configuration
    final_cfg = parse_full_cfg(parser, argv)
    return final_cfg
```

### 场景：养生至尊

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/unit9/Health-Gathering-Supreme.png" alt="Health-Gathering-Supreme"/>



该场景的目标是 **教智能体如何在不知道是什么让它生存的情况下生存**。代理只知道这一点 **生命是宝贵的** 死亡是不好的所以 **它必须了解是什么延长了它的存在，并且它的健康与生存息息相关**.

地图是一个包含墙壁和绿色酸性地板的矩形， **定期伤害玩家**。最初，一些医疗包均匀分布在地图上。时不时会有新的医疗包从天而降。 **医疗包可以治愈玩家部分生命值** - 为了生存，特工需要接他们。该剧集在玩家死亡或超时后结束。

进一步配置：
- 生活奖励 = 1
- 3 个可用按钮：左转、右转、前进
- 1 个可用的游戏变量：健康
- 死刑=100

您可以找到有关 ViZDoom 中可用场景的更多信息 [这里](https://github.com/Farama-Foundation/ViZDoom/tree/master/scenarios).

还有许多为 ViZDoom 创建的更复杂的场景，例如详细介绍的场景 [这个 github 页面](https://github.com/edbeeching/3d_control_deep_rl).
