## 什么是自玩？

在对抗性游戏中正确训练代理可以 **相当复杂**.

一方面，我们需要找到如何让训练有素的对手与你的训练代理对战。另一方面，如果你找到一个训练有素的对手，当对手太强大时，你的智能体将如何改进其策略？

想想一个刚刚开始学习足球的孩子。与非常优秀的足球运动员比赛是没有用的，因为很难获胜，或者至少有时很难拿到球。所以孩子会不断地失败，而没有时间去学习好的政策。

最好的解决方案是 **拥有与代理处于同一级别的对手，并且会随着代理升级自己的级别而升级**。因为如果对手太强，我们就什么也学不到；如果对手太强，我们就什么也学不到。如果它太弱，我们就会在面对更强大的对手时过度学习无用的行为。

这个解决方案称为 *自玩*。在自玩中， **代理使用其自身以前的副本 (其政策的) 作为对手**。这样，代理就会与同等级的代理对战 (有挑战性但不过分)，有机会逐步改进自己的政策，然后随着对手变得更好而更新对手。这是一种引导对手并逐渐增加对手复杂性的方法。

这与人类在竞争中学习的方式相同：

- 我们开始与水平相当的对手进行训练
- 然后我们从中学习，当我们掌握了一些技能后，我们就可以与更强的对手走得更远。

我们对自我游戏也做了同样的事情：

- 我们 **首先以我们代理的副本作为对手** 这样看来，这个对手的水平也差不多了。
- 我们 **从中学习** 并且，当我们掌握了一些技能时，我们 **用我们最新的训练政策副本更新我们的对手**.

自我对弈背后的理论并不是什么新鲜事。它已经被 Arthur Samuel 的跳棋玩家系统在 50 年代使用，并被 Gerald Tesauro 的 TD-Gammon 在 1995 年使用。如果你想了解更多关于自我对弈的历史 [查看安德鲁·科恩 (Andrew Cohen) 撰写的这篇非常好的博文](https://blog.unity.com/technology/training-intelligent-adversaries-using-self-play-with-ml-agents)

## MLAgents 中的自玩

Self-Play 已集成到 MLAgents 库中，并由我们将要研究的多个超参数进行管理。但正如文档中所解释的，主要焦点是 **最终策略的技能水平和通用性与学习稳定性之间的权衡**.

针对一组缓慢变化或不变且多样性较低的对手进行训练 **带来更稳定的训练。但如果变化太慢，就会存在过度拟合的风险。**

所以我们需要控制：

- 如何 **我们经常更换对手** 与 `swap_steps` 和 `team_change` 参数。
- 的 **保存的对手数量** 与 `window` 参数。较大的值 `window`
 意味着代理的对手池将包含更多样化的行为，因为它将包含训练运行早期的策略。
- 的 **与当前自己对战对手的概率** 从池中取样 `play_against_latest_model_ratio`。较大的值 `play_against_latest_model_ratio`
 表示代理将更频繁地与当前对手对战。
- 的 **保存新对手之前的训练步骤数** 与 `save_steps` 参数。较大的值 `save_steps`
 由于该策略接受了更多的训练，因此将产生一组涵盖更广泛技能水平和可能的游戏风格的对手。

要获得有关这些超参数的更多详细信息，您肯定需要 [查看文档的这一部分](https://github.com/Unity-Technologies/ml-agents/blob/develop/docs/Training-Configuration-File.md#self-play)


## 用于评估我们代理的 ELO 分数

### 什么是 ELO 分数？

在对抗性游戏中，跟踪 **累积奖励并不总是跟踪学习进度的有意义的指标：** 因为这个指标是 **只取决于对手的技术。**

相反，我们使用的是 ***ELO评级系统*** (以阿帕德·埃洛 (Arpad Elo) 命名) 计算出 **相对技能水平** 来自给定人群的 2 名玩家之间的零和游戏。

在零和游戏中：一个代理获胜，另一个代理失败。它是每个参与者效用的获得或损失的情况的数学表示 **完全由其他参与者的效用的得失来平衡。** 我们谈论零和游戏，因为效用之和等于零。

这个 ELO (从特定分数开始：通常为 1200) 最初可以减少，但在训练过程中应该逐渐增加。

Elo系统是 **根据对其他玩家的输和平来推断。** 这意味着玩家的评分取决于 **关于对手的评分以及对他们的得分结果。**

Elo 定义了 Elo 分数，即零和游戏中玩家的相对技能。 **我们说相对是因为这取决于对手的表现。**

中心思想是考虑球员的表现 **作为正态分布的随机变量。**

2名玩家之间的评分差异为 **比赛结果的预测因子。** 如果玩家获胜，但获胜的概率很高，那么它只会比对手赢得几分，因为这意味着它比对手强得多。

每场比赛结束后：

- 获胜的玩家获得 **失败者的分数。**
- 点数确定 **根据 2 位玩家评分的差异 (因此是相对的).**
    - 如果评分较高的玩家获胜→评分较低的玩家将被扣掉一些分数。
    - 如果评分较低的玩家获胜→评分较高的玩家会被扣掉很多积分。
    - 如果是平局 → 评分较低的玩家会从评分较高的玩家那里获得一些分数。

因此，如果 A 和 B 的评级为 Ra 和 Rb，则 **预期分数是** 给出：

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/unit10/elo1.png" alt="ELO Score"/>

然后，在游戏结束时，我们需要更新玩家的实际 Elo 分数。我们使用线性调整 **与玩家表现出色或表现不佳的程度成正比。**

我们还定义了每场比赛的最大调整等级：K 因子。

- 对于主控，K=16。
- 对于较弱的玩家，K=32。

### 示例

如果我们举个例子：

玩家A的评分为2600

玩家 B 的评分为 2300

- 我们首先计算期望分数：

\\(E_{A} = \frac{1}{1+10^{(2300-2600)/400}} = 0.849 \\)

\\(E_{B} = \frac{1}{1+10^{(2600-2300)/400}} = 0.151 \\)

- 如果组织者确定 K=16 并且 A 获胜，则新的评级将为：

\\(ELO_A = 2600 + 16*(1-0.849) = 2602 \\)

\\(ELO_B = 2300 + 16*(0-0.151) = 2298 \\)

- 如果组织者确定 K=16 并且 B 获胜，则新的评级将为：

\\(ELO_A = 2600 + 16*(0-0.849) = 2586 \\)

\\(ELO_B = 2300 + 16 *(1-0.151) = 2314 \\)


### 优点

使用 ELO 分数有多种优势：

- 积分是 **总是平衡的** (当出现意外结果时，会交换更多积分，但总和始终相同).
- 它是一个 **自修正系统** 因为如果一个玩家战胜了一个弱玩家，他们只能赢得一些分数。
- 它 **与团队游戏配合使用**：我们计算每个团队的平均值并在 Elo 中使用它。

### 缺点

- 埃洛 **不考虑个人贡献** 团队中的每个人。
- 通货紧缩评级： **良好的评级需要随着时间的推移保持相同的评级技巧**.
- **无法比较历史评分**.
