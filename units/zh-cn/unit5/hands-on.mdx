## SnowballTarget ⛄

如果您需要了解此环境的工作原理，请查看此部分 👉

### 下载环境 zip 文件并将其移至 `./training-envs-executables/linux/`

- 我们的环境可执行文件位于 zip 文件中。
- 我们需要下载它并将其放置到 `./training-envs-executables/linux/`
- 我们使用 Linux 可执行文件，因为我们使用 colab，而 colab 机器操作系统是 Ubuntu (操作系统)

```bash
# Here, we create training-envs-executables and linux
mkdir ./training-envs-executables
mkdir ./training-envs-executables/linux
```

我们从以下位置下载了文件 SnowballTarget.zip https://github.com/huggingface/Snowball-Target 使用 `wget`

```bash
wget "https://github.com/huggingface/Snowball-Target/raw/main/SnowballTarget.zip" -O ./training-envs-executables/linux/SnowballTarget.zip
```

我们解压可执行文件.zip

```bash
unzip -d ./training-envs-executables/linux/ ./training-envs-executables/linux/SnowballTarget.zip
```

确保您的文件可访问

```bash
chmod -R 755 ./training-envs-executables/linux/SnowballTarget
```

### 定义 SnowballTarget 配置文件
- 在 ML-Agents 中，您定义 **在 config.yaml 文件中训练超参数。**

有多个超参数。为了更好地理解它们，您应该阅读中每一项的解释 [文档](https://github.com/Unity-Technologies/ml-agents/blob/release_20_docs/docs/Training-Configuration-File.md)


你需要创建一个 `SnowballTarget.yaml` 配置文件位于 ./content/ml-agents/config/ppo/

我们将为您提供此配置的初步版本 (复制并粘贴到您的 `SnowballTarget.yaml file`), **但你应该修改它**.

```yaml
behaviors:
  SnowballTarget:
    trainer_type: ppo
    summary_freq: 10000
    keep_checkpoints: 10
    checkpoint_interval: 50000
    max_steps: 200000
    time_horizon: 64
    threaded: true
    hyperparameters:
      learning_rate: 0.0003
      learning_rate_schedule: linear
      batch_size: 128
      buffer_size: 2048
      beta: 0.005
      epsilon: 0.2
      lambd: 0.95
      num_epoch: 3
    network_settings:
      normalize: false
      hidden_units: 256
      num_layers: 2
      vis_encode_type: simple
    reward_signals:
      extrinsic:
        gamma: 0.99
        strength: 1.0
```

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/unit7/snowballfight_config2.png" alt="Config SnowballTarget"/>

作为实验，尝试修改一些其他超参数。 Unity 提供非常 [很好的文档在这里解释了它们中的每一个](https://github.com/Unity-Technologies/ml-agents/blob/main/docs/Training-Configuration-File.md).

现在您已经创建了配置文件并了解大多数超参数的作用，我们准备好训练我们的代理了🔥。

### 培训代理

为了训练我们的代理，我们需要 **启动 mlagents-learn 并选择包含环境的可执行文件。**

我们定义四个参数：

1. `mlagents-learn <config>`：超参数配置文件所在路径。
2. `--env`：环境可执行文件所在的位置。
3. `--run_id`：您要为训练运行 ID 指定的名称。
4. `--no-graphics`：在训练期间不启动可视化。

<img src="https://huggingface.co/datasets/huggingface-deep-rl-course/course-images/resolve/main/en/unit7/mlagentslearn.png" alt="MlAgents learn"/>

训练模型并使用 `--resume` 标记在中断时继续训练。

> 如果您使用时，第一次会失败 `--resume`。尝试重新运行该块以绕过该错误。

培训将需要 10 到 35 分钟，具体取决于您的配置。去拿一个☕️你应得的🤗。

```bash
mlagents-learn ./config/ppo/SnowballTarget.yaml --env=./training-envs-executables/linux/SnowballTarget/SnowballTarget --run-id="SnowballTarget1" --no-graphics
```

###  修改PyramidsRND配置文件
  
- 与第一个环境（自定义环境）相反， **Pyramids由Unity团队制作**.
- 因此 PyramidsRND 配置文件已经存在并且位于 ./content/ml-agents/config/ppo/PyramidsRND.yaml 中
- 您可能会问为什么“RND”位于 PyramidsRND 中。 RND 代表 *随机网络蒸馏* 这是产生好奇心奖励的一种方式。如果您想了解更多信息，我们写了一篇文章解释了这项技术： https://medium.com/data-from-the-trenches/curiosity-driven-learning-through-random-network-distillation-488ffd8e5938

对于本次培训，我们将修改一件事：
- 总训练步数超参数太高，因为我们可以达到基准 (平均奖励 = 1.75) 仅 1M 训练步骤。
👉 为此，我们转到 config/ppo/PyramidsRND.yaml，**并将 max_steps 更改为 1000000。**
